# 案件全流程

三阶段一体化：录入案件 → 立案审查+改编号 → 受理/不予受理。依次执行脚本，从PDF扫描到最终归档。

## 执行步骤

### 阶段1：录入案件

1. 提取PDF信息并生成Excel，自动移入ovh/：
```bash
cd $HOME/hermes && $HOME/hermes/venv/bin/python3 extract_and_register.py
```

2. 核对Excel，修正OCR常见误识别（如张贺赘→张贺贺、方皑→方垲、王情→王倩），确认新案号正确：
   - 不予受理：2位数加"10"前缀（46→1046），3位数加"1"前缀
   - 正常复议：编号不变

3. 预检去重（检查案件是否已在系统中存在）：
```bash
cd $HOME/hermes && $HOME/hermes/venv/bin/python3 precheck_duplicates.py
```
如案件刚从系统删除，可跳过此步。

4. 自动录入（跳过已存在的网络申请案件）：
```bash
cd $HOME/hermes && $HOME/hermes/venv/bin/python3 auto_fill_v11.py
```

5. **⏸ 暂停通知用户：案件已录入，需人工审查后移入立案审查阶段，确认后再继续。**

### 阶段2：立案审查 + 改编号

```bash
cd $HOME/hermes && $HOME/hermes/venv/bin/python3 case_review.py
```

逐个案件：搜索 → 点击办理 → caseEdit() → 填表改编号 → 保存

- 重名处理：同名申请人自动递增搜索结果索引（第1个→第2个），并用caseId去重避免操作同一案件
- 不予受理：编号加前缀
- 正常复议：编号不变
- 表单填写：行政管理类别→其他、处理类型→立案受理/不予受理、行政复议请求→撤销并责令重作、受理时间→今天（仅受理案件）

### 阶段3：受理/不予受理

```bash
cd $HOME/hermes && $HOME/hermes/venv/bin/python3 case_accept.py
```

扫描 ovh/ 文件夹，根据文件夹名判断案件类型，自动处理：

**受理：** 搜索 → 办理 → showpage(3) → 填表 → 受理完成 → 移入ovh2/

**不予受理：** 搜索 → 办理 → showpage(4) → 填表+上传决定.pdf → changeCaseState()(type=2) → 直接结案 → 移入ov/

**重复案件：** 自动检测网络申请/来信申请重复，优先受理网络版，来信版编号加"9"前缀标记

**归档：** 受理→ovh2/，不予受理→ov/

## 案号格式规则

| 类型 | 文件夹格式 | 系统案号 |
|------|-----------|---------|
| 受理 | 常府行复（2026）N号 | 常行复〔2026〕N号 |
| 不予受理 | 常府行复不字（2026）N号 | 常行复〔2026〕1N号(3位) / 10N号(2位) |

字符替换：文件夹用"幺"，系统用"么"
