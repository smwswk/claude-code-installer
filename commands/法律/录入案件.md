# 录入案件

从 `/Volumes/ScannedDocuments/要新建` 根目录扫描新案件文件夹，提取 PDF 信息，生成 Excel，自动录入系统，完成后移入 ovh 归档。

## 执行步骤

### 1. 扫描新案件
检查 `/Volumes/ScannedDocuments/要新建/` 根目录下的文件夹（**排除 ov/ 和 ovh/**），列出所有含 决定.pdf 或 终止.pdf 的文件夹。如果没有新案件，告知用户并停止。

### 2. 提取 PDF 信息并生成 Excel
运行提取脚本：
```bash
cd $HOME/hermes && $HOME/hermes/venv/bin/python3 extract_and_register.py
```

此脚本会：
- 用 pdftotext 提取每个 PDF 的文本
- 解析：来信人姓名、性别、出生日期、身份证号、被申请人名称、申请日期、行政行为名称
- 从被申请人名称推断行政管理类别和行政复议事项
- 生成 `$HOME/hermes/案件登记详细信息表.xlsx`
- **自动将已提取的文件夹移入 ovh/**

### 3. 预检去重
运行预检脚本，检查案件是否已在系统中存在：
```bash
cd $HOME/hermes && $HOME/hermes/venv/bin/python3 precheck_duplicates.py
```

此脚本会：
- 登录系统，导航到立案审查列表
- 一次性加载所有已有案件
- 用申请人姓名匹配本地Excel数据
- 在Excel中标记每个案件的预检状态（新增/已存在-网络申请/疑似重复）
- 跳过录入已存在的网络申请案件

### 4. 检查并补全 Excel
检查生成的 Excel，对提取失败的字段（常见问题：申请日期缺失、被申请人未识别）手动补全。特别注意：
- **街道办事处**类的被申请人（如常福街道办事处）在 searchDeptName API 中搜不到，需要手动处理
- 答复书（非决定书）的 PDF 提取可能不完整
- 日期缺失时需要从 PDF 全文确认

### 4. 运行自动录入（已存在案件自动跳过）
```bash
cd $HOME/hermes && $HOME/hermes/venv/bin/python3 auto_fill_v11.py
```

录入脚本会自动：
- 登录系统（验证码用 ddddocr OCR 识别）
- 逐个导航到表单页面
- 填写全部必填项（日期、姓名、被申请人种类+名称、行政管理类别/子类别、行政复议事项、行政处罚子项、行政行为名称）
- 被申请人名称通过 searchDeptName API 搜索后从下拉框选择（解决"不能粘贴"问题）
- 公安案件自动选"其他"子类别，行政处罚案件自动选"其他"子项
- 点击"登记完成"
- 验证 dept_uuid 是否设置（确认被申请人正确选中）

### 5. 确认结果
检查每个案件的验证输出，确认 dept_uuid 已设置。未设置的（如街道办事处类）需提醒用户手动补录。

## 关键文件
- 提取脚本：`$HOME/hermes/extract_and_register.py`
- 预检脚本：`$HOME/hermes/precheck_duplicates.py`
- 录入脚本：`$HOME/hermes/auto_fill_v11.py`
- Excel：`$HOME/hermes/案件登记详细信息表.xlsx`
- 截图：`$HOME/hermes/screenshots/`

## 已知限制
- 街道办事处在系统搜索 API 中搜不到，dept_uuid 无法自动设置
- OCR 提取的日期有时会误匹配出生日期，需人工核对
- 答复书格式的 PDF（非决定书/终止书）提取信息不完整
- 预检匹配仅用姓名，同名不同案可能误标（需人工确认"疑似重复"）
