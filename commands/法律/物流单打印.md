# 物流单打印

案件诉讼要打物流单(EMS 邮寄凭证)时用。**半自动**:用户日常 Chrome 操作,Python 后置打印(绕 EMS 反爬)。

## 执行

```bash
$HOME/hermes/venv/bin/python3 \
  $HOME/.cc-switch/skills/ems-print-workflow/print_ems_waybills.py \
  $ARGUMENTS
```

参数:
- 直接给单号: `物流单打印 1259321709912 1259323160912`
- 从文件: `物流单打印 --from-file ~/Desktop/waybills.txt`
- 多份: `--copies 2`
- 不打印只看 PDF: `--no-print`

## 流程(用户做 + 脚本做)

**1. 脚本启动**:
- `pbcopy` 单号到剪贴板(逗号分隔)
- `open -a "Google Chrome"` 打开 EMS 查询页

**2. 用户在 Chrome 里**(已登录、已装 LogisticsEnhancer):
- 点击 EMS 输入框
- 跑书签栏的 JS bookmarklet 注入单号
- 点蓝色"查询"按钮
- 点扩展注入的绿色"运行"按钮(扩展开始批量截图,落盘到 ~/Downloads)

**3. 脚本继续**:
- 轮询 ~/Downloads 等所有 EMS_<单号>.png 落盘
- 移动到 ~/Downloads/ems_waybills/
- magick 合 PDF
- lp 打印到 HP M233sdn

## 已知卡点

- **JS bookmarklet 在书签栏**:用户已有,不在的话 SKILL.md 里有完整代码
- **扩展默认下载到 ~/Downloads**:如果用户改过 Chrome 下载位置,脚本会找不到 PNG
- **超时 10 分钟**:扩展抓取慢/卡的话,脚本会超时
