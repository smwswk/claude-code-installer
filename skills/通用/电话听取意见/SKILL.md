# 电话听取意见

## 触发
用户提到以下任一关键词即激活本 skill：
- "电话听取意见"
- "听取意见告知书"
- "办案电话记录"
- "告知书"
- "邮寄电话记录"
- "批量记录"

## 模板位置
```
/Volumes/ScannedDocuments/陈科听取意见书/
├── 听取意见权利告知书2.docx     # 告知书模板
├── 新版邮寄电话记录.docx         # 单案叙事式电话记录模板
└── 电话批量记录.xls              # 批量汇总表格模板
```

## 输入信息（每案一行）
用户会提供案件基本信息，例如：
```
常府行复【2026】137号    顾孝军    尚湖镇    不服履职申请答复书
```
列对应关系：
1. 案号（如：常府行复【2026】137号）
2. 申请人/第三人（抬头，可同申请人）
3. 被申请人全称（如：常熟市尚湖镇人民政府）
4. 决定名称（如：履职申请答复书）

以及每案的地址和电话（用户会给或留空）。

## 输出文件
工作目录：`~/Documents/cases_YYYYMMDD/`

每案生成：
- `告知书-{案号后缀}.docx` — 听取意见权利告知书
- `新版邮寄电话记录-{案号后缀}.docx` — 单案叙事式电话记录

整体一份：
- `电话批量记录.xlsx` — 批量汇总表格

## 工作流程

### 1. 收集案件信息
逐行解析用户提供的信息，确认：
- 案号
- 申请人（= 第三人/抬头）
- 被申请人全称
- 决定名称
- 地址
- 电话

### 2. 生成告知书（docx）
使用模板 `听取意见权利告知书2.docx`，替换占位符：
- `«案号»` → 实际案号
- `«第三人（抬头，可同申请人»` → 申请人名
- `«申请人»` → 申请人名
- `«被申请人（全称）»` → 被申请人全称
- `«决定名称»` → 决定名称
- `2026年3月30日` → 当天日期

**关键技术**：
- 必须用 **XML 字符串替换**（解压 docx → 改 document.xml → 重新打包）
- **禁止**用 python-docx 的 `paragraph.text = ''` 后重建段落，会丢失原始格式

**案号右对齐**：
找到案号段落，显式设置 `WD_ALIGN_PARAGRAPH.RIGHT`，并去除前面空格。

### 3. 生成电话记录（docx）
使用模板 `新版邮寄电话记录.docx`，在 XML 层面做字符串替换：
- 模板中的具体值（如"浙江天勤建设有限公司"、"{{PHONE}}"等）替换为当前案件信息
- 每案一份，分开保存

**格式保留**：必须用 XML 替换，保留原始段落格式（字体大小、行距、缩进等）。

### 4. 统一字体为 SimSun
在 XML 层面处理，**禁止**用 python-docx 的 `run.font.name`（会只改部分 runs）。

用正则替换所有 `w:rFonts` 标签：
```python
import re
xml = re.sub(
    r'<w:rFonts[^/]*/>',
    '<w:rFonts w:ascii="SimSun" w:hAnsi="SimSun" w:eastAsia="SimSun"/>',
    xml
)
```

处理范围：
- `word/document.xml`
- `word/styles.xml`

**原因**：原始模板中 rFonts 标签混乱（"宋体"、"宋体 (中文正文)"、未指定字体混用），会导致 LibreOffice 转换 PDF 时混用 SimSun + Gulim + PingFangSC 三种字体，打印出来大小不一。

### 5. 生成电话批量记录（xlsx）
模板 `电话批量记录.xls` 用 soffice 转 xlsx：
```bash
soffice --headless --convert-to xlsx 电话批量记录.xls
```

然后用 openpyxl 编辑，每案一行：
- 听取时间
- 第三人
- 申请人电话
- 被申请人（全称）
- 案号
- 地址
- 申请人
- 决定名称

### 6. 转换为 PDF
```bash
soffice --headless --convert-to pdf *.docx
```

### 7. 打印
检查默认打印机：
```bash
lpstat -p -d
```

提交打印任务（M233sdn__537444_ 为示例，以实际默认打印机为准）：
```bash
# 告知书每案各 2 份
lp -d <printer> -n 2 -o media=A4 告知书-XXX.pdf

# 电话记录每案各 1 份
lp -d <printer> -n 1 -o media=A4 新版邮寄电话记录-XXX.pdf
```

打印机默认 ColorModel=Gray（黑白），Duplex=None（单面），无需额外指定。

## 关键注意事项

1. **XML 替换 vs python-docx 重建**：电话记录必须用 XML 字符串替换保留格式。python-docx 重建段落会导致"字太偏下"等格式问题。
2. **字体统一**：必须在 XML 层面统一 rFonts 为 SimSun，不能只改部分 runs。
3. **案号对齐**：告知书案号必须显式设置右对齐，否则长案号会换行到第二行。
4. **分开保存**：电话记录每个案子单独一份 docx，不要合并到一个文档里。
5. **xls 转 xlsx**：批量记录模板是 .xls 老格式，需先转 xlsx 再用 openpyxl 编辑。
