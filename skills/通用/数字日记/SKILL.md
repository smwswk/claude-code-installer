# 日记生成助手

从系统各数据源采集一天的数字足迹，生成结构化日记 + 叙事总结。

## 触发条件

- 用户说"日记"、"写日记"、"生成日记"、"今天日记"
- 用户提到"数字足迹"、"浏览记录"、"一天总结"

## 执行步骤

### 1. 数据采集

运行采集脚本：

```bash
python3 ~/diary/collect.py [日期]
```

- 日期格式 `YYYY-MM-DD`，不传则默认今天
- 脚本会采集：Safari/Chrome 浏览历史、Claude Code 交互记录、Git 提交、Apple Notes
- 输出 `~/diary/{日期}.md`（结构化日记）和 `~/diary/{日期}.json`（原始数据）

### 2. 读取原始数据

读取 `~/diary/{日期}.json`，获取完整的浏览会话、Claude 交互、主题分类等结构化数据。

### 3. 生成叙事日记

基于数据生成一段叙事性日记，附加到 `~/diary/{日期}.md` 末尾的 `## 叙事日记` 部分下。

**叙事风格要求：**
- 简洁、个人化、有温度，像写给自己的日记而非报告
- 按早中晚的时间线叙事，描述关键活动而非罗列每条记录
- 提炼 Claude 协作的核心成果（哪几件事做成了、哪几件事卡住了）
- 融入搜索内容反映的兴趣和关注点
- 隐私内容（脚本已标记为[私密浏览]）只需一笔带过，不展开
- 结尾用一句话总结今天的调性

**叙事模板参考：**

```
今天是从[起始状态]开始的。[1-2句概括今天的核心节奏]

[上午段落：主要活动 + Claude协作重点]

[下午段落：主要活动 + 进展]

[傍晚/晚间段落：放松/兴趣/搜索的内容]

今天 [N] 次 Claude 交互，主要集中在[任务A]和[任务B]，[一句话总结一天的调性]。
```

### 4. 完成确认

告诉用户日记已生成，给出文件路径。

## 注意事项

- 如果采集脚本报错（如 Safari DB 被锁），提示用户关闭浏览器后重试
- 叙事中不要出现原始 URL、技术路径（如 /Volumes/...），用自然语言描述
- 如果当天数据为空，生成一条"今天似乎很安静"的短日记
- 支持 `python3 ~/diary/collect.py 2026-04-28` 查看历史日期
