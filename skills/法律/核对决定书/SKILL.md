# 核对决定书

审阅法律文书（行政复议决定书、答复书等），以公文级别标准检查文本错误，输出带 Word 批注的版本供用户确认。

## 触发条件

- 用户说"核对决定书"、"审阅文书"、"核实文书"
- 用户指定一个目录，要求检查其中的 .docx 法律文书
- 默认目录：`/Volumes/ScannedDocuments/核对决定书/`

## 审阅标准（公文级别）

逐段通读全文，按以下维度逐一检查：

### 1. 法条引用准确性
- 法律名称是否完整、正确（含书名号）
- 条款号是否与引用内容匹配（尤其注意法律修订后的条款号变化）
- 常见陷阱：2022年修订的《妇女权益保障法》、2023年修订的《行政复议法》条款号全部重新编排
- 核实方法：凭知识库交叉验证，不确定时标注"待核实"

### 2. 法律概念与逻辑
- 是否存在概念混淆（如"宅基地使用权"vs"房屋所有权"）
- 法律推理链条是否完整
- 引用法条是否支持其论点

### 3. 语法与用词
- 介词搭配（"因/为/由/经"等）
- 动宾搭配是否合理
- 是否有口语化表述（公文应用书面语）
- 长句是否缺少句子成分

### 4. 称谓一致性
- 全文中当事人称谓是否前后统一
- 正式文书中不应混用"答复人/被申请人"等异名
- 行政机关是否为规范全称

### 5. 标点符号
- 书名号嵌套：外层《》，内层〈〉（U+3008/U+3009）
- 引号规范使用
- 分号、顿号、逗号的正确使用

### 6. 数字与日期
- 年代表述是否规范（公文建议"20世纪90年代"非"1990年代"）
- 日期格式是否全文统一
- 案号、文号格式是否正确

### 7. 空格与排版
- 数字与量词间多余空格（"3 间"→"3间"）
- 介词后多余空格（"于 2025年"→"于2025年"）
- 段落缩进、空行是否统一
- 标题层次是否一致

### 8. 术语一致性
- 全文术语是否统一（如"案涉房屋"vs"涉案房屋"不可混用）
- 同一概念全文用同一表述

### 9. 事实一致性
- 前后段落对同一事实的表述是否一致（如P33查明部分 vs P38认为部分）
- 后文新增修饰词（如"口头"）是否与前文事实认定矛盾

## 执行步骤

### Step 1: 扫描目录

```bash
ls -la "<用户指定目录>"
```

找到所有 .docx 文件。如只有一个文件，直接进入审阅；多个文件逐一处理。

### Step 2: 提取全文

```bash
python3 -c "
import docx
doc = docx.Document('<文件路径>')
for i, p in enumerate(doc.paragraphs):
    text = p.text.strip()
    if text:
        print(f'[P{i}] {text}')
"
```

同时输出段落索引号，方便后续定位。

### Step 3: 逐段审阅

按审阅标准逐段检查，记录每处问题的：
- 段落索引（P序号）
- 定位关键词（在段落中用于搜索的精确文本，尽量短且唯一）
- 问题类型和说明
- 建议修改方案

### Step 4: 生成批注 JSON

将审阅意见整理为 JSON：

```json
[
  {"p_idx": 17, "search": "第五十五条", "comment": "【法条序号错误】...建议改为第五十六条。"},
  {"p_idx": 10, "search": "因实质性化解", "comment": "【介词不当】...建议改为..."}
]
```

`search` 字段必须能在目标段落中精确匹配到文本。

### Step 5: 生成批注版 Word

```bash
python3 ~/.claude/skills/核对决定书/add_comments.py \
  "<原文件>.docx" \
  "<原文件>_批注版.docx" \
  "<comments>.json"
```

### Step 6: 输出

```bash
open "<原文件>_批注版.docx"        # 用默认应用打开
open -R "<原文件>_批注版.docx"     # Finder 揭示文件位置
```

以表格形式向用户汇报审阅结果摘要（严重度/位置/类型/说明），等待确认。

## 严重度分级

| 级别 | 标准 | 示例 |
|------|------|------|
| **关键** | 法律引用错误，影响决定合法性 | 法条序号错、适用已废止法律 |
| **重要** | 语法/用词错误，影响公文严肃性 | 介词搭配不当、关键概念表述歧义 |
| 中等 | 称谓不一致、格式不统一 | "答复人"混用"被申请人" |
| 轻微 | 标点、年代格式等可商榷项 | 半角括号、1990年代 |

## 注意事项

- 批注定位文本（search）要从段落原文中截取，不用改写后的文字
- 如果 search 文本在段落中出现多次，脚本只匹配第一次出现
- 默认批注作者为"审阅人"，日期为当天
- 完成审阅后主动询问是否需要复核
- 如有多份文书，逐份处理，每份独立输出批注版
- **法条核实保守原则**：不确定的条款号标注"待核实"，不要强行拆分或推定；尤其新修订法律（2022妇女权益保障法、2023行政复议法）条款号全部重排，引文可能跨条合并，不要过度分析
- 发现新问题类型后，应主动更新本 skill 的审阅标准维度
