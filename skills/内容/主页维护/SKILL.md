# 主页维护

当用户完成新项目、部署新工具、产出新图片或搭建新工作流时，主动询问是否刷新主页展示。

## 触发条件

以下任一情况发生时自动触发：

1. **新项目上线**：用户部署了新页面/工具到 {{GITHUB_USERNAME}}.github.io
2. **新出图**：用户产出新的 AI 影像/摄影作品（婚纱样片、小说影像化、Sweet Water 等）
3. **新工作流**：用户搭建了新的 AI 自动化工作流
4. **主动询问**：用户主动说"更新主页"、"刷新主页"

## 主页结构

文件位置：`$HOME/Documents/GitHub/{{GITHUB_USERNAME}}.github.io/index.html`

当前主页包含以下区块：

### Hero
- 标题、简介、三个按钮（Gallery / About / Podcast）

### AI 工作流搭建（4 卡位）
当前展示：案件自动化、视觉模型集成、日记备份系统、提醒事项批处理

### 成果展示（4 卡位）
当前展示：婚纱样片 AI (链接)、AI 需求调研 (链接)、播客名片 (链接)、小说影像化

## 工作流程

1. **识别变更**：用户完成了什么？是新项目、新图片、还是新工作流？
2. **问用户**：用 AskUserQuestion 问是否要更新主页，列出具体要更新的内容
3. **更新页面**：
   - 修改 `index.html` 中对应 card 的内容
   - 如有新链接，更新 `href`
   - 如有新项目超过 4 个，考虑替换优先级较低的旧项目
4. **部署**：
   ```
   cd $HOME/Documents/GitHub/{{GITHUB_USERNAME}}.github.io
   git add index.html
   git commit -m "Update homepage: [简述]"
   git push
   ```

## 照片更新

如果用户出了新图，除了主页可能更新外，还需要提醒：

- Photo Gallery 是否有新照片需要同步？（运行 sync_photos.sh + hugo）
- 婚纱样片 Portfolio 是否需要更新？
- 小红书是否要发布？

## 项目优先级参考

成果展示位有限（4个），当用户有超过 4 个可展示项目时，按以下优先级排列：
1. 有独立页面的部署项目（带链接 > 无链接）
2. 近期完成的新项目
3. 对外展示价值高的项目（婚纱、播客 > 内部工具）

## 注意事项

- 只问不推，等用户确认后再改
- 每次更新后 git push，提醒用户 GitHub Pages 有 1-2 分钟延迟
- 如果只是小改动（改个数字/文字），直接改不用问
