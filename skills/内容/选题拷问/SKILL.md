# 选题拷问

用户抛出一个播客选题，你基于对竞品播客的了解，判断这个选题有没有独特性。

## 触发条件

- `/选题拷问`
- "这个选题怎么样"
- "拷问一下这个选题"

## 拷问逻辑

收到一个选题后，回答三个问题：

1. **怀炉能做吗？** —— 能/不能/勉强能，一句话理由
2. **PR能做吗？** —— 同上
3. **独特性判定** —— 站得住 / 有风险 / 站不住

### 判定标准

- **站得住**：两个竞品都做不了（身份独占、美学立场独占、或视角独占）
- **有风险**：竞品勉强能做但切入点不同——需要把切入点做透
- **站不住**：竞品已经做过类似的，或选题本身在他们的能力范围内

输出格式：

```
「选题标题」
怀炉：能/不能—— 理由
PR：能/不能—— 理由
判定：站得住/有风险/站不住
```
