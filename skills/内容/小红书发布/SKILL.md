## 能力

### 1. 查看发布队列
用户说"还有啥待发""进度怎么样了""给我看看队列"时：
- 展示已发列表（带发布日期，如已知）
- 展示待发队列（按推荐顺序）
- 展示 extras 储备池
- 标注每个待发内容的标签/话题

### 2. 推荐下一个发布
用户说"接下来发什么""帮我选一个"时：
- 按待发队列顺序推荐
- 给出推荐理由（流量话题匹配度、内容类型差异、避免同质化）
- 提醒发布规则（每天最多3组，07压轴单独发）

### 3. 记录新发布
用户说"XX已发""标记为已发布"时：
- 将对应内容从"待发"移到"已发"
- 更新 `project_xiaohongshu_progress.md` 中的进度
- 提醒剩余待发数量和 extras 储备

### 4. 新增储备
用户说"我拍了新的""新增一个系列"时：
- 询问：故事线名称、预计帧数、主话题标签
- 添加到 extras 储备列表
- 更新 memory

### 5. 发布策略建议
用户说"帮我规划本周发布"时：
- 根据待发队列 + extras 储备，给出周发布计划
- 考虑：话题多样性、发布时间节奏、测试新赛道 vs 稳定输出

### 6. 发布前质检（内容审查门）

用户贴 desc/文案过来说"发布前过一遍"、"check 一下"、"看看这条行不行"、"质检"时触发。
**不要直接说"挺好的"或"可以发"**，必须跑完三段质检再给结论。

#### 6.1 HKR 选题质检（适配小红书）

- **H (Hook，吸睛)**：第一行有钩子吗？读者会停下还是划走？
- **K (Knowledge，信息量)**：读者能带走什么？技法 / 视角 / 情绪锚点 / 推荐？空空一片 = K 不及格
- **R (Resonance，共鸣)**：能不能让读者觉得"这就是我"或"我也有过这种感觉"？

S 级三项兼备；及格至少 2 项；只占 1 项 → **建议改稿**

#### 6.2 L4 活人感四维（核心审查层）

借自 khazix-writer 的活人感终审框架，小红书短内容版：

- **温度感**：是体感记忆（"那天风把外套吹得鼓鼓的"）还是知识性描述（"我感到很治愈"）？后者是 AI 味
- **独特性**：换一个摄影博主能不能写出几乎一样的文案？能 → 独特性不及格
- **姿态**：是"我跟朋友聊"还是"博主向粉丝输出"？后者读者本能屏蔽
- **心流**：从头读到尾有没有哪句话卡住、需要重读？有 → 那句要重写

#### 6.3 禁用词 + 套话扫描

**摄影圈套话**（命中即标）：光与影的诗意 / 用镜头记录 / 按下快门的那一刻 / 凝固时间 / 用心去感受

**AI 味通用踩雷**（来自卡兹克禁用清单）：说白了 / 本质上 / 换句话说 / 不可否认 / 意味着什么 / 这意味着

**已经烂大街的小红书词**（建议替换或删除）：治愈系 / 氛围感 / 心动了 / yyds / 绝绝子 / 姐妹们（除非真的在喊姐妹）

**标点检查**：双引号 `""` 改 `「」` 或不加；破折号 `——` 改逗号/句号；冒号可保留但问一下"真的需要吗"

#### 6.5 不要做的事

- 不要直接帮用户重写文案（除非用户明说"你帮我改一版"）
- 不要给空泛建议（"可以再生动一些"），必须指到具体句子
- 不要因为礼貌而放水——质检的价值在于敢说"这条不能发"
- 短内容可能 3 行就质检完，那就 3 行，别为了显得专业凑长

## 注意事项
- 发布进度以 memory `project_xiaohongshu_progress.md` 为准，每次对话前先读取最新状态
- 如果用户提到具体图片文件或目录，用 `ls` 检查实际文件是否存在
- 发布规则是硬性约束（每天最多3组），计划时必须遵守
- 07 "漫长的告别"是全新赛道，建议单独发测试流量反应，不要和其他内容打包
